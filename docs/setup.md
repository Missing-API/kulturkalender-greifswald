# Setup Checklist

This document turns architecture and stack decisions into an actionable setup plan.

## Goal

Make the repository runnable and verifiable with minimal but complete scaffolding.

## 1. Project Bootstrap (Required)

Create the baseline runtime files:

- `package.json`
- `pnpm-lock.yaml`
- `tsconfig.json`
- `next.config.ts` or `next.config.js`
- `next-env.d.ts`
- `.gitignore`
- `.env.example`

## 2. Source Structure (Required)

Create the initial module tree from [architecture.md](architecture.md):

- `src/contracts/`
- `src/services/application/`
- `src/services/adapters/kulturkalender/`
- `src/services/shared/`
- `src/infrastructure/`
- `src/types/`
- `app/api/v1/`

## 3. Schema-First Foundation (Required)

Implement first zod schemas before business logic:

1. Source payload schema (`kulturkalender.source.schema.ts`)
2. Normalized event schema (`normalized-event.schema.ts`)
3. Public API DTO schemas (`events.schemas.ts`)

Use strict schemas, descriptions, defaults, and deterministic transforms as specified in [architecture.md](architecture.md).

## 4. Quality Baseline (Required)

Set up baseline quality tooling:

- ESLint using `@schafevormfenster/eslint-config`
- Type check: `tsc --noEmit`
- Tests: Vitest configuration
- Report output configuration to `/reports` (XML/JUnit, JSON, HTML)

Create scripts in `package.json` for:

- `dev`
- `build`
- `start`
- `lint`
- `typecheck`
- `test`

Create report folders and ensure they are generated by test tooling:

- `/reports/tests/`
- `/reports/coverage/`

## 5. CI/CD Baseline (Required)

Add minimal CI/CD as defined in [tech-stack.md](tech-stack.md):

- `.github/workflows/ci.yml` for PR checks (`lint`, `typecheck`, `test`)
- Vercel Git integration for deploy on `main`

## 6. Runtime Concerns (Required)

Implement baseline support for:

- Caching: source and venue lookup caching with TTL/stale behavior
- Logging: structured logs with request id, source URL, durations, failures
- Error taxonomy: 400 validation, 5xx upstream/internal categories

Use these canonical specs:

- [specs/caching.md](specs/caching.md)
- [specs/logging.md](specs/logging.md)
- [specs/testing.md](specs/testing.md)

## 7. Data-Source Specific Setup (Required)

Implement first adapter capabilities:

- Feed fetch from configured source URL
- Venue lookup and HTML parsing flow
- Bounded batching and retry strategy

Follow [data-source.md](data-source.md) and [specs/proven-adapter-conventions.md](specs/proven-adapter-conventions.md).

## 8. Definition of Done for Initial Setup

Initial setup is complete when:

1. `pnpm install` works.
2. `pnpm lint`, `pnpm typecheck`, and `pnpm test` run locally.
3. CI runs these checks on pull requests.
4. Deployment to Vercel is connected and triggered from `main`.
5. First source payload can be parsed and validated through zod schemas.
